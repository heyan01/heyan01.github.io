React基本写法总结

  • 从进公司一来一直在用react写app运营后台,主要用途是给本公司的运营人员或外公司的兼职人员使用,对app上的展示文章、视频或其他素材进行管理,编辑等;选用的样式框架是蚂蚁金服开发的ant design;好了,废话不多说,开始上代码;

  • react的核心是虚拟DOM和组件化,这样做的好处就是避免了获取元素带来的速度缓慢,以及代码管理方便和可复用;
    react的语法支持jsx, 这种语法厉害了,js代码和html代码可以写在一起,用{}包起来,里面写js代码,如果碰到尖括号()自动解析成标签;

  • react.js 是 React 的核心库
  • react-dom.js 是提供与DOM相关的功能,会在window下增加ReactDOM属性
  • browser.js 的作用是将JSX语法转为JavaScript语法

公司项目中的react使用

首先利用ant init生成项目的脚手架,项目基本的架子就搭好了;路由用的是前端路由,使用react-router;
把所有的组件即页面文件用jsx文件书写, 每个文件里创建一个或多个组件,最终利用export default Component返回一个写好的组件;在App.jsx文件里汇总,根据路径不同,展示不同的组件,最后返回App,
入口文件ReactDom.render(<Routes>,document.getElementById('root')),这里的Routes,是react路由文件,判断不同的路径加载不同的组件;基本是这样子的,细拆有很多小的点,这次先写到这。
