CSS3动画

css3 2D转换,可以对元素进行移动(translate),缩放(scale,),转动(rotate),拉长或拉伸(skew);

Internet Explorer 10、Firefox 以及 Opera 支持 transform 属性。
Chrome 和 Safari 需要前缀 -webkit-。
Internet Explorer 9 需要前缀 -ms-。

例如:顺时针旋转30度;

1 translate 方法;从原始位置移动元素; 参数分别是left(水平方向)偏移量和top(垂直方向)偏移量;若相同,可以写一个参数

2 rotate 方法:元素顺时针旋转给定的角度。允许负值，元素将逆时针旋转。写法见开篇;

3 scale() 方法:缩放元素;宽度变为原始尺寸的2倍,长度变为原始尺寸的1倍,可以只有一个参数,表示宽度和长度的缩放比例一致;

4 skew 倾斜方法; 围绕 X 轴把元素倾斜翻转 20 度(正值就是向后仰,然后从盒子左下角看)，围绕 Y 轴倾斜翻转30度(正值是盒子右侧往前翻,然后从盒子左下角看)。这个属性不太好理解,看图更易懂。楼主贴了3张图辅助理解;

x轴翻转 transform:skew(30deg)

y轴翻转 transform:skew(0,10deg)

transform:skew(30deg,10deg)

5 matrix() 混合方法;把所有的2D方法组合在一起;需要六个参数，包含数学函数，允许您：旋转、缩放、倾斜以移动及元素。
注意：translate, rotate等方法都是需要单位的，而matrix方法参数的单位可以省略。
matrix(cosθ,sinθ,-sinθ,cosθ,0,0) 最后两个参数表示水平和垂直的位移量;前四个参数表示旋转的角度,但是需要将角度0换算为余弦和正弦;明显不如rotate来得简单直观; 感觉这个混合属性被用到的场景不会太多;
有关matrix属性深入理解 移步

css3 3D变换(transform); 也可以移动(translateX,translateY,translateZ,translate3d),缩放(scaleX,scaleY,scaleZ,scale3d),旋转(rotateX,rotateY,rotateZ,rotate3d),视距(transform:perspective(100px)),3D透视(transform-style: preserve-3d),不面对屏幕是否可见(backface-visibility)

  1. rotateX,类似向后翻单杠效果;

rotateY,类似翻书效果;但是轴是在书页中间

rotateZ,沿着Z轴顺时针翻转,

rotate3d(x,y,z,angle)

2.translateZ() 必须与同时设置perspective(),当tranlateZ(199)时,元素充满整个屏幕,一叶障目不见泰山的效果,如果大于等于200则元素消失,即眼睛无法看到身后的东西;

实际应用,旋转木马效果: 需要一个舞台和容器

舞台 需要加视距 perspective: 800px;
容器 加个3D视图声明 transform-style: preserve-3d;

前端插件综述

npm:是Node.js的包管理工具（package manager）。

为啥我们需要一个包管理工具呢？因为我们在Node.js上开发时，会用到很多别人写的JavaScript代码。如果我们要使用别人写的某个包，每次都根据名称搜索一下官方网站，下载代码，解压，再使用，非常繁琐。于是一个集中管理的工具应运而生：大家都把自己开发的模块打包后放到npm官网上，如果要使用，直接通过npm安装就可以直接用，npm install angular(实际上就是从NPM官网上下载下来angular这个包,并放到node modules这个文件夹里)不用管代码存在哪，应该从哪下载。

更重要的是，如果我们要使用模块A，而模块A又依赖于模块B，模块B又依赖于模块X和模块Y，npm可以根据依赖关系，把所有依赖的包都下载下来并管理起来。否则，靠我们自己手动管理，肯定既麻烦又容易出错。
任何时候都可以直接删除整个node_modules目录，因为用npm install命令可以完整地重新下载所有依赖。并且，这个目录不应该被放入版本控制中。就是不能上传到git上。

模块:在Node环境中，一个.js文件就称之为一个模块（module）。

使用模块可以避免函数名和变量名冲突(原理是闭包实现)。编写代码不必从零开始;模块的名字就是文件名（去掉.js后缀）,所以如果引用模块 就写
require(./hello) 写模块的相对路径;如果只写模块名,不写相对路径,会依次去内置模块、全局模块和当前模块下查找hello.js

common JS:模块加载机制被称为CommonJS规范。
一个模块想要对外暴露变量（函数也是变量），可以用module.exports = variable;，一个模块要引用其他模块暴露的变量，用var ref = require(‘module_name’);就拿到了引用模块的变量。

小程序之学习笔记

  • 微信小程序的到来,使那些使用频率不高,功能单一的原生App(不叫APP,全世界只有中国人叫APP,人家是application的简写)闻风丧胆。尽快掌握小程序对于开发者而言,百利而无一害;下面开始进入正题;

全局文件

* 具有通用型的全局文件有三种类型,app.js, app.json, app.wxss三个,

app.js添加公用的方法

  • App() 函数用来注册一个小程序。接受一个 object 参数，小程序的生命周期函数放在对象里,还可以存放任何一个开发者自定义的函数。在其他页面的.js文件里获取自定义的函数

    onShow 小程序显示就会触发;onHide小程序隐藏就会触发;onError监听错误 有错误才触发;

app.json 里配置app,决定页面文件的路径、窗口表现、设置网络超时时间、设置多 tab 等。

app.wxss 放置公共的样式,写法与普通样式写法无异

当打开控制台,会发现,每个页面的最外层html结构都是<page></page>,发现并不是在.html文档里写入,而是因为在每个页面对应的那个js文件里都需要调用一个Page()函数,接受一个对象参数,函数的作用正式注册一个页面,指定页面的初始数据、生命周期函数、事件处理函数等。所以在页面结构中最外层就是<page></page>,先写到这,以后补充;
