小程序之学习笔记

  • 微信小程序的到来,使那些使用频率不高,功能单一的原生App(不叫APP,全世界只有中国人叫APP,人家是application的简写)闻风丧胆。尽快掌握小程序对于开发者而言,百利而无一害;下面开始进入正题;

全局文件

* 具有通用型的全局文件有三种类型,app.js, app.json, app.wxss三个,

app.js添加公用的方法

  • App() 函数用来注册一个小程序。接受一个 object 参数，小程序的生命周期函数放在对象里,还可以存放任何一个开发者自定义的函数。在其他页面的.js文件里获取自定义的函数

    onShow 小程序显示就会触发;onHide小程序隐藏就会触发;onError监听错误 有错误才触发;

app.json 里配置app,决定页面文件的路径、窗口表现、设置网络超时时间、设置多 tab 等。

app.wxss 放置公共的样式,写法与普通样式写法无异

当打开控制台,会发现,每个页面的最外层html结构都是<page></page>,发现并不是在.html文档里写入,而是因为在每个页面对应的那个js文件里都需要调用一个Page()函数,接受一个对象参数,函数的作用正式注册一个页面,指定页面的初始数据、生命周期函数、事件处理函数等。所以在页面结构中最外层就是<page></page>,先写到这,以后补充;

React基本写法总结

  • 从进公司一来一直在用react写app运营后台,主要用途是给本公司的运营人员或外公司的兼职人员使用,对app上的展示文章、视频或其他素材进行管理,编辑等;选用的样式框架是蚂蚁金服开发的ant design;好了,废话不多说,开始上代码;

  • react的核心是虚拟DOM和组件化,这样做的好处就是避免了获取元素带来的速度缓慢,以及代码管理方便和可复用;
    react的语法支持jsx, 这种语法厉害了,js代码和html代码可以写在一起,用{}包起来,里面写js代码,如果碰到尖括号()自动解析成标签;

    基本写法 在index.js文件里要这样写组件

然后在后缀是 .html文件里,需要加载几个脚本

  • react.js 是 React 的核心库
  • react-dom.js 是提供与DOM相关的功能,会在window下增加ReactDOM属性
  • browser.js 的作用是将JSX语法转为JavaScript语法

公司项目中的react使用

首先利用ant init生成项目的脚手架,项目基本的架子就搭好了;路由用的是前端路由,使用react-router;
把所有的组件即页面文件用jsx文件书写, 每个文件里创建一个或多个组件,最终利用export default Component返回一个写好的组件;在App.jsx文件里汇总,根据路径不同,展示不同的组件,最后返回App,
